## 📦 Project Dependencies Breakdown

### 🚀 Core Framework & Next.js

The foundation of the application, managing rendering, routing, and component state.

- **`next`** (`16.2.7`) — React framework for production-grade web applications.
- **`react`** (`19.2.4`) — Core library for building user interfaces.
- **`react-dom`** (`19.2.4`) — Entry point to the DOM and server renderers for React.

### 🎨 UI Core & Styling (Shadcn / Tailwind)

Libraries responsible for design, layout consistency, and interactive UI primitives.

- **`shadcn`** (`^4.11.0`) — Component library built on Radix primitives.
- **`radix-ui`** (`1.5.0`) — Unstyled, accessible component primitives for design systems.
- **`lucide-react`** (`^1.17.0`) — Clean and consistent icon library.
- **`next-themes`** (`^0.4.6`) — Dark mode and theme management for Next.js.
- **`postcss`** (`^8.5.15`) — Tool for transforming CSS with JavaScript plugins.
- **`clsx`** (`^2.1.1`) / **`tailwind-merge`** (`^3.6.0`) — Utilities for efficiently constructing and merging Tailwind CSS classes conditionally.
- **`class-variance-authority`** (`^0.7.1`) — Manages variant-driven component states easily.
- **`tw-animate-css`** (`^1.4.0`) — Extended animation capabilities.

### 🖼️ Image Studio

Engines and decoders handling client-side image processing.

- **`@imagemagick/magick-wasm`** (`^0.0.40`) — WebAssembly port of ImageMagick for advanced format conversions.
- **`heic2any`** (`^0.0.4`) — Client-side conversion of HEIC/HEIF images to JPEG or PNG.

### 📝 Character Studio

Engines, parsers, and sanitizers handling client-side text processing, analysis, and validation.

- **`marked`** (`^12.0.0`) — A high-performance Markdown compiler for parsing markup into structured text layouts.
- **`dompurify`** (`^3.0.9`) — DOM-based XSS sanitizer for cleaning HTML and safe character rendering.

### Data Studio

### 📊 Data Studio

Libraries responsible for parsing, reading, and transforming structured data formats directly in the browser.

- **`js-yaml`** (`^5.2.1`) — YAML parser and serializer used to read, validate, and convert YAML configuration files into JavaScript objects (and vice versa).
- **`xlsx`** (`^0.18.5`) — Spreadsheet processing library for reading, writing, and converting Microsoft Excel (`.xlsx`, `.xls`) and other tabular data formats such as CSV directly in the browser.

### 🎬 Video & Audio Studio

Engines utilized for media decoding, cutting, and transcoding directly in the browser.

- **`@ffmpeg/ffmpeg`** (`^0.12.15`) — WebAssembly video/audio processing suite.
- **`@ffmpeg/util`** (`^0.12.2`) — Core helpers and utilities for managing ffmpeg execution.

### 🛠️ Shared File Handling & Utilities

Under-the-hood utilities for zip compression, feedback alerts, and local file management.

- **`jszip`** (`^3.10.1`) — Creating, reading and editing `.zip` files for batch downloads.
- **`file-saver`** (`^2.0.5`) — Client-side saving/downloading of processed files.
- **`react-hot-toast`** (`^2.6.0`) — Lightweight, responsive notifications for system feedback.

### 📧 Email & Contact System

Backend configurations and templating engines for handling transactional emails.

- **`resend`** (`^6.12.4`) — Email API platform for sending operational messages.
- **`@react-email/components`** (`^1.0.12`) — Pre-built React components for clean email design.
- **`@react-email/render`** (`^2.0.9`) — Utility to translate React components into HTML email strings.
